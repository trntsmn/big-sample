Bit Src
Share Components between projects and team members

From the Horse's Mouth

Bit is a component-as-a-service platform that helps software developers to organize the components of their codebase, make them discoverable, sync them between different projects and use them to build new features and products faster. With Bit, you can focus on product and technological innovation instead of having to reinvent the wheel.

Bit is a collaborative open source project, actively developed and maintained by a venture-backed team and used by more teams and communities every day.

Installing Bit

npm install bit-bin --global
yarn global add bit-bin

Check out Bit Docs for even more options.

Initializing Bit

bit init

As you will see, Bit is very similar to Git source control.

Ready Components for Export

Bit has a straight-forward process as far as exporting components to your desired scope. While simple, it does involve a number of steps and an adjustment in workflow. You will have to create a Bit Src account before starting this phase.

Head over to BitSrc.io to start an account.

Adding a Component

To start tracking a component, you need to add it.

bit add src/foo/bar.js

Here are all your options:

bit add|a <files> [-i|--id <id>] [-m|--main <main file name>] [-t|--tests <tests>] [-n|--namespace <namespace>] [-e|--exclude <files to exclude>] [-o|--override]

Adding Multiple Components

Add several components using a file list or add all components sharing a directory & sub-directories.

bit add src/foo/bar.js src/utils/connect.js
bit add src/foo/*
bit add src/foo/**

Add a component that stores the entire directory.

bit add src/foo/

Declaring Name on Add

Use the --id tag to store under a desired namespace.

bit add src/foo/bar.js --id what/ever

Check out Bit Adding for info on adding components.

Checking Bit Status

To see which stage your components are at, run status.

bit status
new components
     > foo/bar... ok
--------------

Component Environments

To build, test, and render components you need to import the proper environment specs.

Here's an example using Webpack & React.

bit import bit.envs/bundlers/webpack -c
bit import bit.envs/compilers/react -c

Check out Bit Envs for more options.

Build & Tag Your Component

Run build to ready your component.

bit build foo/bar

After the build, give your component a version tag if you are happy with it. You can specify the version or run it for all tracked components if you want.

bit tag foo/bar
bit tag foo/bar 1.0.2
bit tag --all

Staged Components

If we run status again, we will see our component is staged and ready for export.

bit status
staged components
     > foo/bar. versions: 0.0.1... ok
--------------

Exporting Components

It has been a workout getting here, but we can now export our components. If you haven't started a Bit Src account yet, it is absolutely necessary at this point. So what are you still sitting on your hands for?

Head over to BitSrc.io to start an account.

Let's Export!

Once you create a new scope thru your Bit Src account, you can export your components to it.

bit export username.scope

If you check the status again, you will see it is empty

bit status
* there's nothing to see here *

Previewing Components

Storing components in an online scope has its perks but the real benefit is being able to preview. You can see working versions of your components right in your component library on BitSrc.io, pretty nifty indeed. Utilizing the preview mode does require one more step though, so let's check out an example of getting a preview setup.

Setting Up for Preview

At the top of your component file, add some commented text to create preview specs.

/**
* @render react
* @name Bar
* @example
* <Bar
*   title="Here is some sample prop info"
*   description="Add whatever here to get a preview" />
*/

Preview of the Preview

Learn more about this great feature at Bit Rendering.

Importing Components to Your Project

As you may have noticed, a lot of the work happens in the staging phase. Actually exporting your component is quite simple and once a component is stored in a scope you can import it anywhere you like, dependencies intact.

Create Scoped Registry

To configure npm installing Bit components, you will need to configure BitSrc.io as a Scoped Registry.

npm config set @bit:registry https://node.bitsrc.io

Installing & Importing Components

You can install any component utilizing the following pattern:

npm install @bit/username.scope.foo.bar

The npm installs dependencies if needed and now you can import the component into your project.

import Bar from '@bit/username.scope.foo.bar';

The End

That's it as far as simple introductions go. If you dig deeper you will find that you can edit components and they will get tagged with a new version as you go. Similar to Git, you can always see what the component looked like in earlier versions. There is a lot more to learn for additional levels of control, so check Bit Docs for further learning.
